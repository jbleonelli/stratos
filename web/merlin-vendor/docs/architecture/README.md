## When to add here

Architecture-of-the-system docs: data model, RLS, cross-cutting policy, new tracks that span multiple PRs. If it answers "how is this designed?" or "why this shape?", it belongs here.

Single-feature playbooks belong in [`../guides/`](../guides/). Canonical lookup tables (the role enum values, the Meridian device fleet) belong in [`../reference/`](../reference/).
