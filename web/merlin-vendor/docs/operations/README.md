## When to add here

Operational concerns — runbooks for on-call procedures, multi-phase rollout tracking, production-readiness checklists, incident response templates. If the doc describes something you'd want during an outage or before a major launch, it belongs here.

How-to content for normal everyday use goes in [`../guides/`](../guides/). System-design rationale goes in [`../architecture/`](../architecture/).
